# 02_lab020107_R: Параллельное программирование в R

## Описание проекта
Данный проект содержит реализацию параллельного программирования на языке R с использованием пакета parallel. В рамках лабораторной работы реализованы два задания, демонстрирующие основные принципы работы с вычислительными кластерами и распределением задач между процессами.


## Задания

### Задание 1: Анализ частоты слов в книгах Джейн Остин
- Использованы данные пакета `janeaustenr` с текстами книг Джейн Остин
- Реализованы функции для извлечения и анализа слов
- Создан вектор максимальной частоты слов по буквам алфавита
- Построена визуализация результатов с помощью `barplot()`
- Использованы пакеты `stringr` и `parallel`

### Задание 2: Параллелизация вычислений
- Создан вычислительный кластер с использованием пакета `parallel`
- Реализована функция генерации случайных чисел и вычисления среднего
- Проведено сравнение последовательного и параллельного выполнения
- Использованы функции `makeCluster()`, `clusterExport()`, `parLapply()`
- Демонстрируется модель Master-worker

## Требования
- R версии 4.0.0 или выше
- Пакет `parallel` (входит в базовую поставку R)
- Пакет `janeaustenr` (для задания 1)
- Пакет `stringr` (для задания 1)

## Примеры использования
*Задание 1: Работа с текстовыми данными*

```r
# Анализ частоты слов для конкретной буквы
max_frequency("a", words_vector, min_length = 5)

all_words <- janeausten_words()
```
*Задание 2: Работа с кластером*

```r
# Создание кластера с 4 ядрами
cl <- makeCluster(4)
clusterExport(cl, "my_function")
results <- parLapply(cl, 1:100, my_function)
stopCluster(cl)
```

## Принципы параллельного программирования, использованные в проекте
1. **Разделение на задачи - независимое выполнение одинаковых операций**
2. **Модель Master-worker - главный процесс распределяет задачи между рабочими**
3. **Разделенная память - каждый процесс работает со своей областью памяти**
4. **Масштабируемость - возможность использования multiple ядер процессора**
5. **Синхронизация - корректное завершение работы всех процессов**
