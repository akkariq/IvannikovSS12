```bash
# Слияние веток

# Отправка на GitHub

```

### Теоретическая основа

## Основные понятия Git
- **Репозиторий** - хранилище проекта и его истории изменений
- **Коммит** - фиксация изменений в репозитории с уникальным идентификатором
- **Индекс** (Staging Area) - промежуточная область для изменений перед коммитом
- **Ветка** (Branch) - изолированная линия разработки

## Статусы файлов в Git

- **untracked** - неотслеживаемый
- **modified** - измененный
- **staged** - подготовленный к коммиту

## Основные команды Git
`git init` - инициализация репозитория
`git status` - состояние рабочего каталога
`git add` - добавление файлов в индекс
`git commit -m "message"` - создание коммита
`git branch <branch_name>` - создание ветки
`git checkout <branch_name>` - переключение на ветку
`git merge <branch_name>` - слияние веток
`git remote add origin `- привязка к удаленному репозиторию
`git push -u origin <branch_name>` - отправка изменений на GitHub
